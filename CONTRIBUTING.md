# Contributing Guide

Thank you to all developers and supporters who contribute to this project!

If you like this project, feel free to give it a Star or contribute!

## Bug Reports

1. Please use the latest version to confirm the bug has not already been fixed.

2. Please confirm the bug is not related to your setup (e.g., outdated browsers like IE, disabled browser features like cookies/storage).

3. Please clearly describe the bug so we can fix it effectively.

4. You can use the "Bug Report" Issue template, but please fill it in correctly.

## Feature Suggestions

1. Please use the latest version to confirm the suggestion has not already been implemented.

2. Please clearly describe your suggestion.

3. You can use the "Feature Request" Issue template, but please fill it in correctly.

## Submitting Code

Please follow the coding standards described below and in desktop.html.

1. Please try to include all changes in a single commit. You can add follow-up commits, but try not to exceed 5.

2. Please use Git CLI, GitHub Desktop, or https://github.dev for submissions. Do not upload files directly through the browser.

3. Please write meaningful commit titles and descriptions, for example:
   - Good: "Fix issue where X does not work", "Add new X application"
   - Bad: "stuff", "forgot this", "too many bugs...qwq"

4. Formatting:
   - Do not use formatting tools on HTML files
   - For JavaScript and CSS files, you may use VS Code's built-in formatter

### Commit Message Format

1. For significant updates:

   ```
   v11.4.5 - Updated X

   (Update by @Someone)
   - Updated...
   - Optimized...
   ```

2. For small fixes, a brief description is sufficient.
